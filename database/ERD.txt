================================================================================
                              RELATIONSHIPS
================================================================================

1. app_users ─────► soil_data
   Relationship: ONE-to-MANY
   Description: One farmer can record multiple soil tests
   
2. app_users ─────► recommendations  
   Relationship: ONE-to-MANY
   Description: One user can receive multiple crop recommendations

3. crops ────────► recommendations
   Relationship: ONE-to-MANY
   Description: One crop can be recommended to many users

================================================================================
                              KEYS SUMMARY
================================================================================

Primary Keys (PK):
  • app_users.id
  • crops.id
  • soil_data.id
  • weather_data.id
  • recommendations.id

Foreign Keys (FK):
  • soil_data.user_id → app_users.id (ON DELETE CASCADE)
  • recommendations.user_id → app_users.id (ON DELETE CASCADE)
  • recommendations.crop_id → crops.id (ON DELETE CASCADE)

Unique Keys:
  • app_users.email
  • crops.crop_name
